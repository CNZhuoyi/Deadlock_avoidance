# 死锁避免演示

## 关于本仓库

一个大二学生的操作系统原理课程设计，顺手传上来（没啥营养）



## 主要要求

银行家算法是具有代表性的死锁避免算法。在避免死锁方法中允许进程动态地申请资源，但系统在进行资源分配之前，应先计算此次分配资源的安全性，若分配不会导致系统进入不安全状态，则分配，否则等待。

1. 初始化完成相关基础数据录入并进行展示。其中，基础数据包括资源种类、可用资源数、进程数、进程最大需求、已分配资源等等。
2. 进程提出请求，系统进行检查，若安全则给出所有的安全序列。
3. 报错提示包括：进程请求资源超过系统资源、进程请求资源超过最大需求、预分配导致系统进入不安全状态等等。
